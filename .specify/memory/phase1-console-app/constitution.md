## **Overview**

This constitution defines the rules, standards, and expectations for **Phase I: In-Memory Python Console Application** of the *Todo App* project.
The purpose of this phase is to build a **command-line todo application** using **Spec-Driven Development** with **Claude Code** and **Spec-Kit Plus**.

The application will store all tasks **in memory**, without any database or file storage.
This foundational console app will evolve into a full AI-native, cloud-native system in later phases.

---

## **1. Core Principles**

### **1.1 Spec-Driven Development**

* All implementation **must be generated by Claude Code**.
* The developer (me) is **not allowed to write code manually**.
* All features must first be written as **specifications** using Spec-Kit Plus.
* Claude Code must strictly follow:

  * The specifications
  * This constitution
  * The required project structure

### **1.2 Clean Code Guidelines**

* Code must be modular, readable, well-structured.
* Clear function names and separation of concerns are required.
* No long, monolithic files ("god files").

### **1.3 In-Memory Data Only**

* Task data must be stored **in RAM only** (e.g., Python lists/dictionaries).
* When the application exits, all tasks must be cleared.
* No files, no JSON storage, no database.

### **1.4 Progressive Evolution**

* This phase establishes the base layer for all future phases.
* All files and structure must be designed with continued growth in mind.

## **3. Non-Functional Requirements**

### **3.1 Terminal User Interface**

* Clear, readable console output.
* Friendly error messages.
* Input validation required (no crashes).

### **3.2 Project Structure**

The project must follow this structure:

```

/specs/                   # All feature specifications (Spec-Kit Plus)
/src/                     # Python implementation (generated by Claude Code)
    main.py
    models.py
    services.py
```

### **3.3 Code Quality**

* Consistent formatting & naming.
* Modularity (models, services, utilities separated).
* No unused variables or dead code.
* Error handling required.
